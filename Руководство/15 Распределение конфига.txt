Если конфиг очень большой, то его желательно распределить по нескольким файлам. Вот некоторые из способов это сделать:

1) Если какую-то задачу можно оформить в виде отдельного, максимально обособленного класса, сделайте это. Например, Вы можете создать класс БазаПредметов, реализовать в нем методы для сохранения характеристик предметов и поиска нужного предмета по какому-либо фильтру, а затем пользоваться этим классом в конфиге.

2) Можно распределять класс по нескольким файлам с помощью ключевого слова partial. К примеру, метод Интерпретировать класса Каркас в клиенте вынесен в отдельный файл Каркас\Каркас.Интерпретировать.cs.

3) Удобно использовать наследование. Точно так же, как класс Окно наследуется от класса Каркас, Вы можете использовать класс Каркас в качестве базового для своего класса, а класс Окно наследовать от Вашего класса. Понятно, что класс Окно должен быть последним в цепочке наследований, так как экземпляры именно этого класса создаются в классе Конструктор. Не забывайте только вызывать методы базового класса для обработки тех данных, которые Вы не обрабатываете в текущем.
